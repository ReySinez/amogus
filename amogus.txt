1.untracked, unmodified, modified, staged
2.Команда git add додає вміст робочої директорії в індекс (staging area) для подальшого комміта.
3.Команда git status показує стану файлів в робочій директорії і індексі: які файли змінені, але не додано в індекс; які очікують коммітов в індексі. Додатково до цього виводяться підказки про те, як змінити стан файлів.
4.Команда git commit бере всі дані, додані в індекс за допомогою git add, і зберігає їх зліпок у внутрішній базі даних, а потім зрушує покажчик поточної гілки на цей зліпок.
5.Ініціалізація Git у папці проекту.
6.Описуйте, що було змінено, де змінено, чому змінено.
7.Команда git checkout використовується для перемикання гілок і вивантаження їх вмісту в робочу директорію.
8.Команда git reset - це складний універсальний інструмент для скасування змін. Вона має три основні форми виклику, відповідні аргументам командного рядка --soft, --mixed, --hard.
9.Команда для перегляду історії комітів.
10.Команда git status показує стану файлів в робочій директорії і індексі: які файли змінені, але не додано в індекс; які очікують коммітов в індексі. Додатково до цього виводяться підказки про те, як змінити стан файлів.
11. Восстановление удаленного файла, если он уже добавлен в индекс (git add): git reset -- myfile
    Восстановление удаленного файла или отмена изменений: git checkout -- myfile
